This document contains the Lumin SDK release notes including the notes for the core SDK, Unity® integration, and Unreal Engine integration.

C API

Features

  • Introduces MLConnections APIs to send and receive invites to the people following the user in the Social App.
  • Introduces MLContactsRequestSelection API to request a subset of contacts as manually selected by the user.
  • Introduces MLMovements APIs for spatial movement of objects to feel like they have mass and inertia by tilting and swaying during movement, but still respecting the user's desired placement. Use this to be consistent with the movements in the Universe.
  • Introduces MLPersistentCoordinateFramesGetFrameState to get additional state data that includes the confidence in the Persistent Coordinate Frame within the valid radius and the rotation and translation error.
  • Introduces new callback in the MLLifecycle to let the application know when it loses or regains focus. When focus is lost, the application will not receive input events.
  • Introduces MLLocationGetLastFineLocation to fetch the last known fine location. The API also provides the accuracy of the location.
  • MLEyeTrackingStaticData now provides the left and right eye gaze vector.
  • MLHandTrackingState now provides whether the hand is holding a Control or not. This feature will have more improvements in future releases, please refer to known issues section for the current limitations.
  • Introduces MLHandTrackingKeyPoint enum to provide the ordering of the keypoints.
  • MLGraphicsFrameInfo structure is updated to provide a handle to a synchronization object that must be cast to VkSemaphore and used during queue submission of a Vulkan app. GL apps can ignore this member.
  • Introduces new MLAudio APIs to get/set head relative sound and the memory footprint for the loaded audio source.
  • Three additional events added to MLMediaDRMEvenType: MLMediaDRMEvenType_SessionReclaimed, MLMediaDRMEvenType_ExpirationUpdate, MLMediaDRMEvenType_KeysChange.
  • Renamed the MLAudio "Input Stream" and "Output Stream" APIs to "Buffered Input" and "Buffered Output" APIs. The older Stream APIs are deprecated.

Deprecation Notices

  • The MLScreens API and its privilege ScreensProvider are deprecated.
  • The following MLAudio APIs are deprecated:
    • MLAudioCreateSoundWithOutputStream
    • MLAudioGetOutputStreamDefaults
    • MLAudioGetOutputStreamLatency
    • MLAudioGetOutputStreamFramesPlayed
    • MLAudioGetOutputStreamBuffer
    • MLAudioReleaseOutputStreamBuffer
    • MLAudioGetInputStreamDefaults
    • MLAudioGetInputStreamLatency
    • MLAudioGetInputStreamBuffer
    • MLAudioReleaseInputStreamBuffer
  • MLGraphicsVirtualCameraInfoArray is deprecated. This is included in the updated version of MLGraphicsFrameInfo.
  • MLHandTrackingHandState is deprecated and replaced by MLHandTrackingHandStateEx.
  • MLHandTrackingData is deprecated and replaced by MLHandTrackingEx.
  • MLHandTrackingGetData is deprecated and replaced by MLHandTrackingGetDataEx.
  • All Controller 6DOF data exposed by way of the MLInput APIs are deprecated. Use the MLController APIs instead.

Known Issues

  • MLHandTracking can now detect if a hand is holding the Control or not. But the current version can have some false positives. One way to address this issue would be to use the MLControl APIs to check the position of the Control to eliminate the false positives.

Notes

  • MLControllerButton_HomeTap cannot be polled, this button is only available via the callbacks.

Features

  • Improved mabu error reporting when a user override of CC=... or CXX=...results in an error.

Bug Fixes

  • Fixed mabu bug where unknown *.config or *.option files next to the project would fail the load. If these are obviously not mabu files (XML, binary, etc.) they are silently ignored. Otherwise, such files only generate warnings rather than failing to build.
  • Fixed bug in mabu cmake toolchain generation in not passing --sysroot=... in CMAKE_C[XX]_FLAGS.
  • Fixed bug where in some environments, mabu would fail to sign .mpks when there are multiple Python installations on the path.
  • Fixed bug trying to detect toolchains with a plain --version request (this was causing problems in Win32 systems with no Visual Studio, and was also doing unnecessary work).
  • Fixed detection of Apple clang 11.0 and newer.

Lumin Runtime SDK 0.23.0

  • Lumin OS 0.98

Lumin Runtime is a declarative application framework that enables you to author both Landscape and Immersive apps. Landscape apps can run and render simultaneously with other Landscape apps and are contained in a volume called a Prism. Immersive apps suspend all other apps and can utilize your entire space. Lumin Runtime relies on a client/server model to render multiple client applications within the real world. It facilitates application sharing out of the box, allowing application developers to collaborate with multiple users.

The Lumin Runtime SDK adds support for creating Lumin Runtime applications to the Magic Leap Lumin SDK, including APIs, examples and build support.

Features

  • App - added functions to check Wifi / internet status.
  • App - added support for named actions.
  • App - added PCF data getters.
  • Events - added video subtitle events.
  • Consts - added new event source: tablet device.
  • Consts - some gesture flags deprecated.
  • Consts - some led patterns deprecated.
  • Prism - added option to create material resource from fd.
  • Prism - added option to create model with explicit material scope.
  • Prism - added option to create texture resource from image2d.
  • Prism - getPrismController works in js.
  • Prism - added function to create image2d.
  • Privileges - added pcf read, web view and hand mesh privileges.
  • Privileges - found object privilege deprecated.
  • EyeTrackingEventData - isFixationDepthUnconfortable function deprecated.
  • InputHelper - works in js now.
  • LightNode - head pose lighting deprecated.
  • ParticleNode - particle samplers work in js now.
  • VideoNode - added protected content flag setter/getter.
  • Notification - notification sound deprecated.
  • Resource - added protected content flag setter/getter.
  • TextureResource - added texture format getter.
  • KeyboardDefines - added enGb, de, fr language codes, added page codes.
  • UiKitConsts - added system icons for aspect ratio.
  • UiWebView - added functions to enable/disable/query virtual keyboard speech to text.

Lumin Runtime Editor 0.23.0

The Lumin Runtime Editor tool provides you with comprehensive features for building scene graphs for your Lumin Runtime Landscape application. The editor includes line, quad, text, model, and transform nodes with which you can arrange the graphical content of your app. All of the ready-made, animated, 3D UI elements from the Magic Leap UI Kit are also available as node types. These predefined UI nodes and properties enable you to rapidly design and develop apps that share the same design sense as other Lumin Runtime applications.

Features

  • Added support for Sprite nodes.
  • Added additional preview support for dynamic content (blend setup, particles, sprites).
  • Added directories as assets support (particles, sprites).
  • Set better defaults for Audio node.
  • Added import for audio assets (.wav, .pcm, .ogg).
  • The Editor now prompts to restart after changing the MLSDK preference so changes will take effect.
  • The Editor validates MLSDK preference so mismatch is not possible.
  • Can add support for new asset types to old projects.
  • Animations now also play in the Zero Iteration previewer.
  • The Build Project command is easier to find in the Editor.
  • The Editor automatically selects the best graphics interface.
  • Old projects automatically update to get newly supported asset types.

Bug Fixes

  • Fixed potential memory leak in generated code.
  • Better undo support for UITextEdit Text Entry Mode.
  • Fixed playback speed in animation preview.
  • Fixed issues with insertion of nodes when nothing is selected.
  • Fixed potential spurious warnings for material names.
  • Fixed potential blank scene when first opening/resizing viewport.
  • Fixed some asset pipeline build issues.
  • Fixed edited UIPanel attributes not showing up on device.
  • Fixed Zero Iteration previewer menu label was sometimes incorrect.
  • Fixed UIDialog rendering on Mac.
  • Fixed node renaming was triggered when clicking on any part of scene hierarchy.
  • Fixed Use Frame checkbox unsetting UIImage resource.
  • Incompatible FBX files now report as a problem.
  • Fixed vertical compression artifact when resizing content viewport.
  • Fixed previewer bugs.
  • Fixed a bug when sprite textures sometimes only partially render.

Known Issues

  • AudioNode spatial sound properties require default values setting properly. Suggested values are:
    • Distance Properties: Min Distance 1.0, Max Distance 10.0, Roll Off Factor 1.0
    • Radiation Properties: Inner Angle 50.0, Outer Angle 340.0, Outer Gain 1.0, Outer Gain High Frequency 0.08
    • Direct Send Levels: Gain 1.0, Gain Low Frequency 1.0, Gain Mid Frequency 1.0, Gain High Frequency 1.0
    • Room Send Levels: Gain 1.0, Gain Low Frequency 1.0, Gain Mid Frequency 1.0, Gain High Frequency 1.0
  • AudioNode spatial sound properties: Radiation properties missing Outer Gain Mid Frequency and Outer Gain Low Frequency elements.

Unreal Package 0.23.0

  • Lumin SDK Version 0.23.0
  • Unreal Version: Unreal Engine compatible with Lumin SDK 0.23.0
  • Lumin OS 0.98

Magic Leap's hardware and software is in a qualified developer release state right now. Please take a close look at this document since it contains valuable guidance on the current state of Magic Leap's hardware and software to help developers make the best possible use of the platform during our pre-release period. Features

  • 0.23.0 release is based on Unreal Engine 4.23.0.
  • Added MagicLeapConnections plugin to send invitations to the user's Magic Leap connections to initiate a multi-user experience. Apps need to add the SocialConnectionsInvitesAccess privilege to manifest to use this API.
  • Added MagicLeapMovementComponent which makes moving objects feel as if they have mass and inertia by tilting and swaying during movement. The component being moved will respect all blocking collisions. Developers can also choose to have soft collisions (allowing a degree of penetration before hitting an impenetrable core) with overlapping objects.
  • Exposed MagicLeapNetworking plugin which provides functions to check internet connectivity status and wifi data (RSSI, link speed, frequency). Apps need to add the Internet privilege to manifest to use this API.
  • Exposed ARPin translation and rotational errors via FMagicLeapARPinState.
  • Added a boolean parameter (true by default) to SetStabilizationDepthActor() and SetFocusActor() which will set the both StabilizationDepthActor and FocusActor to the same input Actor.
  • Updated bRemoveDebugInfo option in the Magic Leap Project Settings to only affect symbol stripping. The sym file will be generated regardless of this option. Whether the symbols will be stripped from the executable is managed by this.
  • Enabled delayed library loading for the MLSDK libs on all platforms, including Lumin. Calling functions that were not located in the available libraries will return MLResult_NotImplemented instead of just crashing, making Unreal Engine usage more flexible across different MLSDK and LuminOS versions.
  • Added Blueprint function to query if a given ML controller is connected.
  • Added GPU crash marker support for Vulkan on Lumin. Set r.GPUCrashDebugging and r.gpucrash.collectionenable to 1 in the Lumin device profile or in ConsoleVariables.ini to enable GPU crash collection.
  • Exposed a timestamp with the FMLInputTabletDeviceState data.
  • Exposed state change callbacks for Music Service to Blueprints.
  • Unreal examples have been updated to calibrate the Pawn on map load so as to make the app launch position the origin instead of keeping the origin stuck to the device boot location. Check out the XRPawn blueprint for the calibration logic. The setup of MotionControllers, MRMesh, and so on have also been updated to respect the pawn calibration.
  • Reduced frequency of the expensive audio jack update check to free up the audio thread.
  • Updated time consuming UMagicLeapImageTrackerFunctionLibrary::EnableImageTracking() & UMagicLeapImageTrackerFunctionLibrary::SetMaxSimultaneousTargets() functions to be asynchronous.
  • MagicLeapSecureStorage now supports int64, Blueprint Structs, TArrays and USaveGame objects. Functions for storing and retrieving explicit structs (FVector, FTransform etc) have been deprecated and should be replaced with Get/PutSecureArray.
  • MagicleapLocations API functions have a new parameter to choose between getting a fine location and a coarse location. The accuracy of the fine location radius is also provided. The locations integration can also be used by way of the "Mobile Location Services Blueprint Library" plugin interface.
  • Added support for localized app names. Add the language code and corresponding name for the app in the Advanced MPK Packaging section in the Magic Leap Project Settings in the Editor.
  • Added getters functions for hand keypoints to motion source names and vice-versa.
  • ImageTrackerComponent has been moved into it's own module named MagicLeapImageTracker. There is no change required for Blueprints as the appropriate redirectors have been put in place but C++ developers will now need to add this new module as a dependency in their game / plugin module's Build.cs file and include MagicLeapImageTrackerComponent.h header.
  • Added selection functionality to the MagicLeapContacts plugin. Developers can now instigate a system ui popup that displays all of their contacts. The user can then select which contacts they wish to retrieve data as well as filter the fields to be retrieved.
  • Added a template in the Editor which sets up an XRPawn and a "Hello Cube!" map to help jump start development for Magic Leap.

Deprecation Notices

  • To avoid naming conflicts with any other data types defined in the engine or the accompanying plugins, all public data types (enums, structs & classes) in the Magic Leap plugins have been prefixed with 'MagicLeap'. For example, URaycastComponent is now UMagicLeapRaycastComponent, FRaycastHitResult is now FMagicLeapRaycastHitResult and so on. Note that this does not require any changes in any of the existing blueprints as appropriate redirectors have been put in place. C++ usage of these, however, would need to be updated. Please refer to the Unreal Engine API Reference on https://developer.magicleap.com/learn/reference/public/v0.23.0/UnrealAPI/index.html for the complete list of data types.
  • OpenHandBack gesture has been deprecated and its usage should be replaced with OpenHand.
  • PwFoundObjRead privilege has been deprecated and its usage should be replaced with PcfRead.
  • Deprecated SetBasePosition/Orientation/Rotation functions. The XRPawn setup should be used instead to manipulate.
  • The "additional contents" packaging option has been removed from the Magic Leap Project Settings. The regular options under the Engine's Packaging settings should be used.
  • The "additional content binary" packaging option has been removed from the Magic Leap Project Settings. Binaries to be packaged should be added as Runtime Dependencies or Unreal Plugin Language should be used to copy the files over to the bin folder of the staging directory.
  • MagicLeapScreensComponent which was deprecated in MLSDK 0.21.0 has now been deleted.
  • MagicLeapScreens API integration has been marked deprecated and will be removed in the next release.

Known Issues / Disclaimers

  • With every engine upgrade, many new features and tools are added by Epic Games. It is possible that the path for certain files in the engine gets too long for Windows. It is advised to keep the root folder path for the engine installation to the minimum possible to avoid "path too long" errors.
  • ML device does not appear in Session Frontend on Mac.
  • Symbol stripping on the final Lumin executable fails on Mac for Blueprint-only projects when using the Magic Leap UE4 Editor from Epic Games Store (issue not seen in source builds). This error does not fail the actual packaging process itself and can be ignored during development. If you still want the symbols to be stripped, delete the Intermediate/Lumin/Mabu/Binaries folder in your project before triggering the package build. A proper fix is planned in a future MLSDK release.
  • All UBT, UAT operations and anything else that requires Mono fails on Mac when using the Magic Leap UE4 Editor from Epic Games Store (issue not seen in source builds) if MonoFramework is not already installed on the host machine. You can install MonoFramework on your Mac either via MonoDevelop or via Visual Studio for Mac. A proper fix will be provided in the next MLSDK release.
  • Video capture sample fails if Bumper is pressed too frequently.
