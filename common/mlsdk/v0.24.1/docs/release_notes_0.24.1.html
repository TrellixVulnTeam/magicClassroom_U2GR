This document contains the Lumin SDK release notes including the notes for the core SDK, Unity® integration, and Unreal Engine integration.

C API

New Features

  • NOTE: 0.24.1 is a patch fix for Unity's spatial audio MSA plug-in and to mark C API Found Objects API as experimental. Lumin SDK 0.24.0 and 0.24.1 are otherwise identical for other platform areas.
  • Bluetooth Low Energy: Introduces BLE client APIs
    • BLE client app can scan for a BLE peripherals .Get remote RSSI, name, and other device specific information.
    • Connect to GATT server and discover GATT service record from the server.
    • Enable/disable notifications and indications from peripheral devices.
    • Get the state of local Bluetooth adapter (Bluetooth ON/OFF state).
    • Create Bond with peripheral device for an authenticated and encrypted communication between the devices.
  • MLAppConnect: Enables application to get the avatar settings, asset data (meshes, materials, textures), animation data, and material properties (e.g. model opacity, current hair color) from the Avatar Engine so that the Avatar model can be created and rendered from within the Application. Note: This API is meant to be used via other higher level rendering engines. Support for Unity and Unreal engines will be available in future releases.
  • MLAvatar: Enables application to get the avatar settings, asset data (meshed, materials, textures), material properties (e.g. model opacity, current hair color) from the Avatar Engine so that the Avatar model can be rendered from within the Application. Note: This meant to be used via other higher level rendering engines. Support for Unity and Unreal engines will be available in the future release.
  • MLDCA: Enables an ML1 app to subscribe to platform services for file modifications and create networking resources on behalf of a desktop application. This allows for applications on the desktop to be created that can communicate with certain ML1 platform services.
  • MLFoundObject: Enables application to query for objects in the real world. The following 5 object categories will be supported in this release: couch, chair, table, poster, and screen. Each found object will have an associated bounding box and score. This is an experimental feature.
  • MLArucoTracker: Enables applications to track square fiducial markers (also known as Augmented Reality Markers). It allows for rendering virtual content in relation to the location of Aruco markers that can be attached to flat surfaces.
  • MLMediaCea708: Enables application to support CEA708 close captions. The parsed data is provided via asynchronous APIs and is in UTF-8 format.
  • Media Player Sharing: Applications can share the media player across different users.
    • MLMediaPlayerSetSharingInfo and MLMediaPlayerSetID APIs will setup the sharing experience.
    • on_info callback will provide notifications when the shared media players starts, stops, pauses, or seeks.

Updates to existing features

  • MLAudio:
    • Added a polling and callback API to notify when the output mute has changed.
    • Added a callback API to notify when the output device has changed.
    • Added support for 6-channel audio to MLAudioCreateSoundWithBufferedOutput.
    • Added support for 24-bit integer and 32-bit integer/float to MLAudioCreateSound(orResource)WithLoaded(orStreamed)File.
  • MLConnection:
    • Added an API to get the current user's followers, followed, and mutual connections. The returned structure will include the connection's name and personalization settings (which can be used with the MLAvatar API).
    • Added an API to deliver the status of each user invited via the MLConnections API. The status can be one of the following: Processing, Dispatched, Delivered, Failed.
    • Added an API for the initiator to start a session by generating a join-code. Other users can then use this join-code to join the same session. Invitations based on join by code do not provide status of the participants in the session.
  • MLContacts:
    • MLContactsCancelRequest will now return MLResult_Ok on success instead of MLContactsResult_Cancelled.
    • Several MLContact APIs will provide additional return values.
    • Versions of several MLContacts structures were incremented.
  • MLDispatch:
    • Added a new MLDispatch OAuth API MLDispatchOAuthOpenWindowEx, this replaces MLDispatchOAuthOpenWindow. With this new API applications can disallow http urls.
    • Added a new MLDispatch OAuth API, MLDispatchOAuthRegisterSchemaEx, to register for a callback that also provides a context pointer in the callback.
  • MLInput:
    • MLInput APIs provide more granular API specific MLInputResult.
    • on_touchpad_gesture_start callback was renamed by on_touchpad_gesture callback. The older MLInputControllerCallbacks and MLInputSetControllerCallbacks are deprecated and replaced by MLInputControllerCallbacksEx and MLInputSetControllerCallbacksEx.
  • Multi Media:
    • Added a new callback, on_frame_packing, to provide frame packing information.
    • Added a new API, MLMediaPlayerGetMetrics, to get metrics for the current media player. It provides average frame rate, average video bit rate, and average audio bit rate. This is only supported for DASH streams.
    • The on_info callback has additional events for state transitions when the media player started, stopped, paused, or resumed.
    • CEA608 Closed Caption now supports UTF-16 encoding via synchronous and asynchronous mechanism.
      • MLMediaCCParserGetDisplayableEx is the new is synchronous API that returns the parse dayta in UTF-16 format. The older MLMediaCCParserGetDisplayable API is deprecated.
      • MLMediaCCParserParse is an asynchronous API which will return the parsed data via the MLMediaCCParserDisplayChangedCallback, which can be set using MLMediaCCParserSetDisplayChangedCallback.
    • Added an API, MLMediaPlayerResetAsync, to reset the media player asynchronously. The result of the operation will be delivered via the on_reset_complete callback.
    • MLMusicService has an additional return code.
  • MLSnapshot: Added MLSnapshotGetTransformWithDerivatives to get the pose derivatives (i.e. linear and angular velocity and acceleration) of the Control. Getting pose derivatives for other trackers is currently not supported.
  • MLPersistentCoordnateFrames: Added MLPersistentCoordinateFrameQuery API to submit a query filter for fetching PCFs. PCFs can now be filtered based on type and distance from a given point.
  • MLPrivileges: API privileges are now available as a json file for convenience.

mldb

Features

  • new 'ping-response' command makes the device respond to ping requests (ICMP echo). For security reasons, the device does not respond to pings by default. You can now make it respond during the current boot session with this command.

Bug Fixes

  • When an application launch fails due to a missing library, "mldb log" output now contains the name of the missing library. More generally, all logging by libc in debuggable applications now shows up in 'mldb log'.
  • Server process no longer crashes when unplugging a device.
  • Some filesystem commands failed if the given path(s) didn't contain a leading forward slash. The slash is implicit and thus meant to be optional. For some commands, that wasn't the case.
  • Fixed regression in SDK 0.23 mldb server that prevented multiple simultaneous device connections over TCP/IP. Make sure to do 'mldb kill-server' to start use the new (0.24) server.
  • Fixed macOS mldb's inability to have more than four commands running in parallel.
  • Fixed process leak in 'notify' command.

Lumin Runtime SDK 0.24.1

  • Lumin OS 0.98.10

Lumin Runtime is a declarative application framework that enables you to author both Landscape and Immersive apps. Landscape apps can run and render simultaneously with other Landscape apps and are contained in a volume called a Prism. Immersive apps suspend all other apps and can utilize your entire space. Lumin Runtime relies on a client/server model to render multiple client applications within the real world. It facilitates application sharing out of the box, allowing application developers to collaborate with multiple users.

The Lumin Runtime SDK adds support for creating Lumin Runtime applications to the Magic Leap Lumin SDK, including APIs, examples and build support.

Lumin Runtime Editor 0.24.1

The Lumin Runtime Editor tool provides you with comprehensive features for building scene graphs for your Lumin Runtime Landscape application. The editor includes line, quad, text, model, and transform nodes with which you can arrange the graphical content of your app. All of the ready-made, animated, 3D UI elements from the Magic Leap UI Kit are also available as node types. These predefined UI nodes and properties enable you to rapidly design and develop apps that share the same design sense as other Lumin Runtime applications.

Features

  • MagicScript Components
    • Initial editor support for MagicScript components.
    • Visual Editing of scenes.
    • Project sources generated.
    • JSX for scenes dynamically updated as user edits.
    • Event handlers can be added.
    • Build support targets device, ZI, and mobile platforms.
    • Video resources supported for video component.
  • Prefabs
    • Supported for both Lumin RT (c++) and MagicScript.
    • Reusable sub-scenes that can be added to scenes via the editor.
    • Source generated allows user to spawn prefab instances dynamically.
    • Properties of sub-nodes can be exposed via the prefab root node.

Known Issues

  • VS2017 and VSCode fails to build a LRE project with a handler. The user must define their handlers to build successfully.
  • MagicScript projects created using the editor will have a 'resources' dir that comes from the default MagicScript template, but the editor will put assets into the 'pipeline' dir when added to the asset library. Developers may still use the included 'resources' dir for their own purposes as this will be copied into the application.
  • Quad component creation is disabled on MagicScript projects.
  • Code generation for MagicScript projects may fail on windows for large or multiple scenes.
  • Prefabs in MagicScript projects containing components with the external property selected will run into issues when adding handlers, or when adding the prefab to another scene. As a workaround, make properties public instead of making the entire component or prefab external.

Features

  • 0.24.1 release is based on Unreal Engine 4.24.2.
  • Events for audio muted / unmuted by system have been exposed via the engine's AudioMuteDelegate.
  • Events for audio ducked / unducked by system have been exposed via the engine's UserMusicInterruptDelegate.
  • Exposed functions to get and set mic mute state.
  • Exposed CV camera API as a module of the MagicLeapCamera plugin. Access to the camera intrinsics, pose and texture frames (YUV_420 format only) is available via a BP function library (for more granular control) as well as a component (automatic tracker lifecycle management).
  • Exposed graphics performance numbers provided by the platform via UMagicLeapHMDFunctionLibrary::GetGraphicsClientPerformanceInfo() function.
  • Added ability for developers to set thread affinities by adding appropriate CVars to the Lumin / Lumin_Desktop device profiles or in ConsoleVariables.ini. The CVar lumin.DefaultThreadAffinity sets the thread affinity for Lumin platform with the arguments [MainGame|Rendering|RTHeartBeat|RHI|Pool|TaskGraph|TaskGraphBG|Audio] [int affinity] [optional int affinity2] [optional int affinity3]. e.g. lumin.DefaultThreadAffinity=MainGame 3 4 Rendering 2.
  • Exposed events in LuminLifecycleApplicationComponent to notify app when input focus is lost or regained.
  • Reduced app load out time by optimizing audio buffer flush during app termination.
  • Exposed BP function to check if the Control is being held in a given hand.
  • Exposed BP functions to get current API level of the platform and the minimum API level required by the engine.
  • LAN privilege is requested automatically if app is launched with an IP address in the command-line.
  • Oauth API has been exposed via the MagicLeapDispatch plugin. Triggering an OAuth request will launch the requested auth URL in the Helio browser.
  • LuminARSessionConfig has been updated to expose a FMagicLeapPlanesQuery struct directly to maintain compatibility with the MagicLeapPlanesComponent.
  • Added Visual Studio debugger support for Blueprint-only projects. Run GenerateProjectFiles.bat with the argument "-bponlyproject=<full path to .uproject file>". After that, you can open the generated UE4.sln file, switch the configuration to Lumin & Developement / DebugGame etc and launch the app via the debugger.
  • Exposed callbacks in MagicLeapARPin interface to notify the app when pins are added, updated or deleted. When using this on Lumin, developers can set how frequently the plugin should check for updates and what magnitude of changes in pin state should mark that pin as "updated". These settings are avaialble in Project Settings > Magic Leap AR Pin in the Editor.
  • Added a new member to the MagicLeapARPinState struct to indicate if the Pin can be shared with other devices or is local to the current device.
  • Added a new query function to MagicLeapARPin interface where developers can specify which type of Pins they are interested in (local vs shared) and what volume should these be searched in. This provides a lot more control to filter out the Pins an app would be interested in, reducing the amount of data an app needs to sift through. The interface also allows setting a query filter at the global level which will affect the results reported by the pins added, updated, deleted notification callback. The MagicLeapARPinComponent has also been updated to use this new query filter to search for the most appropriate Pin to bind itself to.
  • MagicLeapARPin interface has been updated to return "NotImplemented" instead of "Unavailable" when it is not implemented on a given platform vs the service being unavailable during that session.
  • Exposed light estimation via the AugmentedReality interface in addition to the already available MagicLeapLightEstimation plugin. Choose "Ambient Light Estimate" in the AR session config to enable light estimation tracking.
  • Exposed image tracking via the AugmentedReality interface in addition to the already available MagicLeapImageTrackerComponent.
  • Updated planes integration in the AugmentedReality interface implementation for Lumin to set "SubsumedByPlane" property to indicate which outer plane a given inner plane belongs to.
  • Updates planes integration in the AugmentedReality interface implementation for Lumin to use delta based queries for a more efficient measure of tracking planes.
  • Magic Leap plugins are now built for iOS as well. While none of the features will be available to use on iOS, having the plugins enabled in your projects will no longer cause building and packaging to fail for either platform.
  • Provide PinToBestFit(), PinToID() and PinToRestoredOrSyncedID() functions in MagicLeapARPinComponent to make it more explicit which PCF will be used for persistence.
  • Expose ObjectUID and UserIndex properties of MagicLeapARPinComponent on spawn i.e. they'll appear in the AddMagicLeapARPinComponent() BP function. This is done to fix the issue where spawning this component at runtime always failed to restore data because BeginPlay() was called before app could set these params. Additionally, made the function UMagicLeapARPinComponent::AttemptPinDataRestoration() public so devs can call it after they set the required ObjectUID.

Deprecation Notices

  • The MagicLeapScreens plugin deprecated in the last release has been removed from the engine. As a consequence, the ScreensProvider privilege, the IsScreensApp option in ProjectSettings, the "Screens" & "ScreensImmersive" component types have also been removed.
  • We missed adding the 'MagicLeap' prefix to the MeshTrackerComponent in the previous release. This has been amended now and the MeshTrackerComponent has been renamed to MagicLeapMeshTrackerComponent. Note that this does not require changes in any of the existing Blueprints as appropriate redirectors have been put in place. C++ usage of this, however, would need to be updated. Please refer to Unreal Engine API Reference on https://developer.magicleap.com/learn/reference/public/v0.23.0/UnrealAPI/index.html for the complete list of data types.
  • Deprecate PinSceneComponent() and PinActor() in MagicLeapARPinComponent and make them work only to pin the self component or the owner actor only. This is important because if we use these functions to pin some random component or actor, we cannot guarantee its restoration and thus persistence because the pinned reference will not mean anything in the next session.

Bug Fixes

  • Any project created with UE 4.23 has the setting r.DefaultFeature.AutoExposure.ExtendDefaultLuminanceRange=True in their DefaultEngine.ini. This setting breaks rendering on Vulkan Mobile path on 4.24 and should be removed.
  • Fixed issue where audio, animations etc would desync from the game when headpose was lost.
  • Fixed hand tracking states not clearing out on consecutive VRPreview launches.
  • Fixed issue where image targets added for image tracking were not deleted when exiting VRPreview.
  • Fixed error destroying input tracking when exiting VRPreview.
  • Fixed issue where next LaunchOn would fail if app is uninstalled outside of the Unreal environment.
  • Fixed issues in destruction of ARPin (PCF) tracker.
  • Fixed bug where symbol stripping would fail for BP-only projects on Mac.
  • Fixed bug where performing any UBT or UAT operations on Mac would fail if MonoFramework is not already installed on the machine.
  • Fixed cook warnings caused by uninitialized data members in the FMagicLeapPlanesQuery struct.
  • Fixed errors in Put/GetSecureArray() functions that caused builds to fail when Blueprint nativization was enabled.
  • Multiple entries for DOF quality were present in the Lumin_Desktop device profile which caused confusion about the default setting. Removed both entries and inherited the value from the parent device profile which disabled DOF by default.
  • Fixed a bug where async versions of the MagicLeapLocation functions were not appearing in the Blueprints.
  • Fixed bug in MagicLeapDispatch module which added the requirement for including the MLSDK module as an additional dependency.
  • r.TranslucentLightingVolume has been re-enabled in the Lumin_Desktop device profile as disabling it was found to be causing an additional 0.4ms on the GPU whereas keeping it enabled and not using translucent lighting volumes in your scene caused so additional cost.
  • Visual Studio debugger used to archive the generated mpk at an odd location, outside of the project folder. Since the debugger picks up the mpk from the project's Binaries folder anyways, this archiving step has been removed when packaging an mpk from within VS.
  • Fixed a bug where the new OpenHand gesture event was never fired and the deprecated OpenHandBack event was still getting reported instead. Note that after this fix OpenHandBack will not be fired and apps will need to replace the deprecated event with the OpenHand event.
  • Fixed log spam of "unspecified failures" from ML function calls when attempting to join a server by passing an IP address in the mldb launch command line. The errors were harmless but no one likes to look at them.
  • Fixed linker errors when using ML plugin data types in C++.
  • Fixed a crash in the editor occurring when "Enable Zero Iteration" was checked in the Project Settings but ZI wasn't actually running.
  • Fixed hand mesh render flickering.
  • Fixed normal calculation for hand mesh.
  • Fixed a bug present in vanilla 4.24 where updating the Magic Leap SDK path in the Project Settings doesn't take effect until the editor is restarted.
  • Fixed planes not working on consecutive VRPReview launches when using the AugmentedReality interface.
  • Fixed MRMesh flickering when using the desktop renderer on UE 4.24.
  • Fixed crashes when starting device stream while the app is actively using the camera.
  • Fixed content persistence when bShouldPinActor in MagicLeapARPinComponent is true.
